# Skill: Defensa de Proyecto DAM

## Cuándo usar este skill
- Cuando prepares el guion o presentación para la defensa final ante el tribunal.
- Cuando necesites justificar decisiones de arquitectura (Clean Arch, MVVM) o tecnologías (Jetpack Compose, Room).
- Para simular preguntas del tribunal y practicar respuestas técnicas.
- En la fase final de Evaluación y Defensa.

## Inputs necesarios
- **Memoria Técnica Final:** Base de la defensa.
- **Proyecto Completado:** Funcionalidades clave a demostrar.
- **Tiempo de Exposición:** (Usualmente 15-20 mins).

## Workflow
1.  **Estructurar la Defensa:**
    - Introducción (Problema/Solución).
    - Demo Técnica (Arquitectura, Código destacado).
    - Demo Funcional (App en vivo).
    - Conclusiones y Futuro.
2.  **Identificar Puntos Fuertes:** Qué destaca técnicamente del proyecto (ej. Uso de Flow, Corucinas, Inyección de Dependencias).
3.  **Preparar Justificaciones:** ¿Por qué Kotlin? ¿Por qué Hilt? ¿Por qué esa base de datos?
4.  **Simular Preguntas:** Generar posibles preguntas difíciles del tribunal y redactar respuestas sólidas.

## Instrucciones
- Mantén un tono profesional, seguro y técnico.
- Evita leer diapositivas; demuestra dominio del código.
- Enfócate en el valor aportado y la complejidad técnica resuelta.
- Prepara una "demo a prueba de fallos" (grabación o flujo muy probado).

## Output (formato guion/preparación)
```markdown
# Preparación de Defensa: [Tema/Sección]

## Puntos Clave a Mencionar
- [Concepto 1]
- [Concepto 2]

## Posibles Preguntas del Tribunal
**Q:** ¿Por qué usaste Room en lugar de Realm o SQLite puro?
**A:** [Respuesta técnica justificada: Abstracción, verificación en tiempo de compilación, integración con Flow...]

## Notas para la Demo
- Recordar mostrar la funcionalidad X para evidenciar la reactividad.
```
